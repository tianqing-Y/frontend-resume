# 贡献指南

感谢你愿意为这个项目贡献！

## 贡献方式

### 提 Issue

遇到以下情况，欢迎提 Issue：

- **识别错误**：某类代码被识别为错误的亮点类型
- **输出格式**：某步骤输出格式不符合预期
- **新亮点类型**：发现 A-F 类之外的隐性亮点规律
- **示例补充**：希望补充某框架或场景的示例

**Issue 模板**：

```
### 问题描述
（一句话描述问题）

### 重现步骤
1. 输入了什么代码
2. 期望得到什么输出
3. 实际得到了什么输出

### 建议方案
（可选）你认为应该怎么改
```

### 提 PR

**PR 类型说明**：

| 类型 | 说明 | 示例 |
|------|------|------|
| `fix` | 修复识别错误或格式问题 | `fix: 修复 F 类亮点识别遗漏 Schema 渲染场景` |
| `feat` | 新增亮点类型或分析维度 | `feat: 新增 G 类亮点「性能监控自建」` |
| `docs` | 文档和示例改进 | `docs: 补充 React 项目使用示例` |
| `example` | 新增完整使用示例 | `example: 新增微前端项目分析示例` |

**PR 流程**：

1. Fork 本仓库
2. 创建分支：`git checkout -b feat/your-feature`
3. 修改 `SKILL.md`（主要文件）
4. 更新 `CHANGELOG.md`（在 `[Unreleased]` 区块记录变更）
5. 提交：`git commit -m "feat: 你的描述"`
6. 推送并创建 PR

## SKILL.md 修改规范

修改时请遵守以下规则，保持文件一致性：

**编号规范**：
- 阶段：Phase 1 / Phase 2 / Phase 3
- 步骤：Step 1 / Step 2 / ... / Step 10
- 小节：1.1 / 1.2 / 5.1 / 5.2 / 8.1 / 8.2 等

**强制标注**：
- 必须执行的步骤：在标题后加 `⛔` 符号
- 有量化价值的亮点：加 `⭐` 符号
- 完成标注格式：`[Phase X · Step Y 完成]`

**优先级标注**：
- P0 必写：🔴
- P1 建议写：🟡
- P2 可选：🟢

## 本地测试

修改 SKILL.md 后，建议用以下方式验证：

1. 在 Claude.ai 的 Project 中粘贴修改后的内容
2. 上传一段真实的前端代码（不含敏感信息）
3. 发送触发词，检查执行流程是否正确

测试用例建议覆盖：
- [ ] 1-3 年用户（初级前端）
- [ ] 有 AI 开发经历的用户
- [ ] 有转型意向的用户
- [ ] 只有口头描述、没有代码的用户
